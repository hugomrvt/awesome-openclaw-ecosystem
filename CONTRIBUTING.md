# Contributing to Awesome OpenClaw Ecosystem

Thank you for your interest in contributing! This list is community-driven and thrives on diverse perspectives.

## Guidelines

### Adding a new project

Please ensure your suggestion meets the following criteria:

- **Relevance**: The project must be related to the OpenClaw ecosystem — a fork, alternative, tool, integration, plugin, deployment method, or educational resource directly connected to OpenClaw.
- **Quality**: The project should have a proper README with documentation.
- **Maintenance**: The project should show activity within the last 6 months.
- **Uniqueness**: Check that the project is not already listed.
- **No spam**: Commercial products are welcome if they provide genuine value to the ecosystem, but self-promotional spam will be rejected.

### How to submit

1. **Fork** this repository.
2. **Create a branch** with a descriptive name: `git checkout -b add-project-name`.
3. **Add your entry** in the appropriate category, following the existing table format.
4. **Ensure** your entry includes:
   - Project name with link
   - Primary language (for code projects)
   - GitHub stars badge (for GitHub-hosted projects)
   - A concise description ending with a period
5. **Submit a Pull Request** with a clear title and description of what you're adding and why it's awesome.

### Format for code projects

```markdown
| [Project Name](https://github.com/user/repo) | Language | ![GitHub stars](https://img.shields.io/github/stars/user/repo?style=flat-square) | Brief description ending with a period. |
```

### Format for services or resources

```markdown
| [Service Name](https://example.com) | Brief description ending with a period. |
```

### Categories

The list is organized into these categories:

| Category | What belongs here |
|----------|-------------------|
| **Official Ecosystem** | Repos under the `openclaw` GitHub organization |
| **Lightweight Assistants** | Minimal footprint, easy to self-host |
| **High-Performance Runtimes** | Built for speed and safety (mostly Rust) |
| **Edge & Embedded** | MCU, ESP32, Raspberry Pi, $5 boards |
| **Forks & Rewrites** | Full reimplementations and language ports |
| **Security & Privacy** | Trust, encryption, sandboxing, red-teaming |
| **Multi-Agent & Orchestration** | Teams of agents, fan-out, workflows |
| **Autonomous & Specialized** | Domain-specific agents |
| **Platforms & Managed Services** | Full-stack solutions, integrated platforms |
| **Hosted OpenClaw** | OpenClaw-as-a-Service providers |
| **Deployment & Infrastructure** | Docker, Helm, cloud, reverse proxy, IaC |
| **MCP Servers & Bridges** | Model Context Protocol integrations |
| **SDKs & Client Libraries** | SDKs, API wrappers, IDE extensions |
| **Plugins & Extensions** | Skills, channel plugins, observability |
| **Tooling, UIs & Dashboards** | GUIs, dashboards, marketplaces, dev tools |
| **Memory & Knowledge** | Long-term memory, knowledge graphs |
| **Voice & Home Automation** | Voice pipelines, Home Assistant bridges |
| **Benchmarks & Research** | Evaluation and testing tools |
| **Agent Social Networks** | Agent-to-agent social platforms |
| **Community Resources** | Curated lists, registries, forums, events |
| **Learning Resources** | Tutorials, videos, podcasts, articles |
| **Adjacent Alternatives** | Enterprise-grade alternatives |

### Suggesting a new category

If your project doesn't fit any existing category, open an [issue](https://github.com/hugomrvt/awesome-openclaw-ecosystem/issues) to discuss creating a new one.

### Removing a project

If you notice a project that is no longer maintained, has been archived, or has broken links, please open an issue or submit a PR to remove it.
